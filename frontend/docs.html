ChartSnap API Documentation

ChartSnap generates high-quality chart images from real market data via a simple REST API — ideal for trading bots, alert systems, portfolio dashboards, and fintech apps.

To get started, register for a free API key. The BASIC tier includes 50 requests/day at no cost.

Version Method Use Case
v1 GET Simple URL-based chart generation — easy to embed anywhere
v2 POST Full JSON config — multiple indicators, styles, save to storage
v3 GET Data endpoints — lookup exchanges and symbols

Authentication 🔒

All API endpoints require authentication. Include your API key using any of these methods:

Method 1 — x-api-key Header (Recommended)

x-api-key: YOUR_API_KEY

Method 2 — Authorization Bearer

Authorization: Bearer YOUR_API_KEY

Method 3 — Query Parameter

GET /v1/tradingview/mini-chart?key=YOUR_API_KEY

Rate Limits ⚠️

Limits are applied per API key, per day. All times reset at midnight UTC.

Tier Daily Limit Rate (req/sec) Max Resolution Max Indicators
BASIC 50 1 800×600 3
PRO 500 10 1920×1080 5
MEGA 1,000 15 1920×1600 10
ULTRA 3,000 35 2048×1920 25

When limits are exceeded, the API returns HTTP 429 with a JSON error body.

Base URL 🔗

http://localhost:3000

All API endpoints are relative to this base URL.

Mini Chart

Capture a compact TradingView mini chart widget — area chart with current price and percentage change.

GET/v1/tradingview/mini-chart

Request Example

cURL
JavaScript
Python
curl "http://localhost:3000/v1/tradingview/mini-chart?symbol=BINANCE:BTCUSDT&interval=1D&theme=dark&key=YOUR_KEY" \
  -o chart.png
const url = 'http://localhost:3000/v1/tradingview/mini-chart'
  + '?symbol=BINANCE:BTCUSDT&interval=1D&theme=dark&key=YOUR_KEY';

const res = await fetch(url);
const blob = await res.blob();
// blob is a PNG image
import requests

url = "http://localhost:3000/v1/tradingview/mini-chart"
params = {"symbol": "BINANCE:BTCUSDT", "interval": "1D", "theme": "dark", "key": "YOUR_KEY"}

res = requests.get(url, params=params)
with open("chart.png", "wb") as f:
    f.write(res.content)

Query Parameters

Parameter Required Default Description
key required — Your API key (or use x-api-key header)
symbol optional BINANCE:BTCUSDT TradingView symbol (EXCHANGE:SYMBOL)
interval optional 1D Chart interval (1m, 5m, 15m, 1h, 4h, 1D, 1W...)
width optional 800 Image width in pixels (tier limited)
height optional 400 Image height in pixels (tier limited)
theme optional dark dark or light

Response

Returns image/png binary data on success.

Advanced Chart v1

Capture a full TradingView advanced chart via GET request. Supports indicators, styles, and themes.

GET/v1/tradingview/advanced-chart

Request Examples

cURL
JavaScript
Python
# Basic candlestick chart
curl "http://localhost:3000/v1/tradingview/advanced-chart?symbol=BINANCE:BTCUSDT&interval=1D&key=YOUR_KEY" -o chart.png

# With RSI + Bollinger Bands
curl "http://localhost:3000/v1/tradingview/advanced-chart?symbol=BINANCE:ETHUSDT&interval=4h\
&studies=Relative+Strength+Index&studies=Bollinger+Bands&key=YOUR_KEY" -o eth.png
const params = new URLSearchParams({
  symbol: 'BINANCE:BTCUSDT',
  interval: '1D',
  theme: 'dark',
  key: 'YOUR_KEY'
});
params.append('studies', 'Relative Strength Index');
params.append('studies', 'MACD');

const res = await fetch(`/v1/tradingview/advanced-chart?${params}`);
const blob = await res.blob();
document.querySelector('img').src = URL.createObjectURL(blob);
import requests

res = requests.get(
    "http://localhost:3000/v1/tradingview/advanced-chart",
    params=[
        ("symbol", "BINANCE:BTCUSDT"),
        ("interval", "1D"),
        ("studies", "Relative Strength Index"),
        ("studies", "MACD"),
        ("key", "YOUR_KEY"),
    ]
)
with open("btc.png", "wb") as f:
    f.write(res.content)

Query Parameters

Parameter Required Default Description
key required — API key
symbol optional BINANCE:BTCUSDT TradingView symbol
interval optional 1D Chart interval
width optional 800 Width in pixels
height optional 600 Height in pixels
style optional candle Chart style (see Chart Styles)
theme optional dark dark or light
studies optional [] Indicator names (repeatable: &studies=RSI&studies=MACD)
timezone optional Etc/UTC TradingView timezone identifier
format optional png png or jpeg

Advanced Chart v2

Enhanced advanced chart via POST with full JSON configuration. Supports indicators with custom inputs, chart drawings, price overrides, and per-study color customization.

POST/v2/tradingview/advanced-chart

Request Header

x-api-key: YOUR_API_KEY
Content-Type: application/json

Request Examples

cURL
JavaScript
Python
curl -X POST "http://localhost:3000/v2/tradingview/advanced-chart" \
  -H "x-api-key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "symbol": "BINANCE:ETHUSDT",
    "interval": "4h",
    "style": "candle",
    "theme": "dark",
    "studies": [
      { "name": "Bollinger Bands" },
      { "name": "MACD" },
      { "name": "Relative Strength Index" }
    ]
  }' -o eth.png
const res = await fetch('/v2/tradingview/advanced-chart', {
  method: 'POST',
  headers: {
    'x-api-key': 'YOUR_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    symbol: 'BINANCE:ETHUSDT',
    interval: '4h',
    style: 'candle',
    theme: 'dark',
    studies: [
      { name: 'MACD' },
      { name: 'Relative Strength Index' }
    ]
  })
});
const blob = await res.blob();
document.querySelector('img').src = URL.createObjectURL(blob);
import requests

res = requests.post(
    "http://localhost:3000/v2/tradingview/advanced-chart",
    headers={"x-api-key": "YOUR_KEY"},
    json={
        "symbol": "BINANCE:ETHUSDT",
        "interval": "4h",
        "studies": [{"name": "MACD"}, {"name": "Relative Strength Index"}]
    }
)
with open("eth.png", "wb") as f:
    f.write(res.content)

Request Body Parameters

Parameter Type Default Description
symbol string BINANCE:BTCUSDT TradingView symbol (EXCHANGE:SYMBOL)
interval string 1D 1m, 5m, 15m, 30m, 1h, 4h, 1D, 1W, 1M, 3M...
width number 800 Width in pixels (tier limited)
height number 600 Height in pixels (tier limited)
style string candle bar, candle, line, area, heikinAshi, hollowCandle, baseline, hiLo, column
theme string dark dark or light
format string png png or jpeg
range string null 1D, 5D, 1M, 3M, 6M, 1Y, 5Y, YTD, ALL
timezone string Etc/UTC IANA timezone identifier
session string regular regular or extended
studies array [] Array of indicator objects (see below)
drawings array [] Array of drawing objects (see Drawings)
override object {} Global chart overrides (colors, labels, etc.)

Studies Array Object

{
  "name": "Bollinger Bands",       // Indicator name (see Indicators list)
  "forceOverlay": false,            // Overlay on main chart pane
  "override": {                     // Per-indicator style overrides
    "Upper.linewidth": 2,
    "Upper.color": "rgb(0,150,255)"
  }
}

Save to Storage

Same as Advanced Chart v2 but saves the image to server storage and returns a URL instead of raw image bytes.

POST/v2/tradingview/advanced-chart/storage

Response

{
  "success": true,
  "filename": "a1b2c3d4-uuid.png",
  "url": "/storage/a1b2c3d4-uuid.png",
  "size": 154320,
  "createdAt": "2026-03-05T06:00:00.000Z"
}

Drawings (v2 POST only)

Add chart drawings via the drawings array parameter in the v2 POST body:

"drawings": [
  {
    "name": "Horizontal Line",
    "input": { "price": 45000 },
    "override": { "lineWidth": 2, "lineColor": "rgb(34,197,94)" }
  },
  {
    "name": "Trend Line",
    "input": {
      "startPrice": 40000, "startBarIndex": -20,
      "endPrice": 48000,   "endBarIndex": 0
    }
  }
]

Supported drawing names: Horizontal Line, Vertical Line, Trend Line, Rectangle, Fib Retracement, Order Line, Long Position, Short Position, Arrow Mark Up, Arrow Mark Down, Text, Callout.

Error Codes

HTTP Code Meaning Solution
200 Success —
400 Bad Request Check your parameters (e.g., missing exchange)
401 Unauthorized Provide a valid API key
404 Not Found Endpoint doesn't exist or exchange not found
429 Too Many Requests Daily or rate limit exceeded — wait or upgrade
500 Server Error Chart generation failed — check symbol validity

Error Response Body

{
  "error": true,
  "message": "Daily request limit of 50 reached for BASIC tier.",
  "tier": "BASIC",
  "dailyLimit": 50,
  "usedToday": 50
}

Register (Get API Key)

Use this endpoint to generate a new API key programmatically.

POST/api/register
curl -X POST "http://localhost:3000/api/register" \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","tier":"BASIC"}'

Response

{
  "success": true,
  "apiKey": "cs-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "email": "you@example.com",
  "tier": "BASIC"
}
