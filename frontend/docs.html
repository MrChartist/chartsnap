Getting Started

ChartSnap API

Generate high-quality trading chart images programmatically. Request PNG/JPEG screenshots of any chart — crypto, stocks, indices — via a simple REST API. No browser required.

💡
Base URL: https://api.chartsnap.io  ·  All endpoints require an API key.  ·  All responses are application/json unless returning a binary image.
Quick start — generate a BTC/USDT daily chart in one line:
curl "https://api.chartsnap.io/v1/tradingview/advanced-chart?symbol=BINANCE:BTCUSDT&interval=1D&key=YOUR_KEY" \
     --output btc_chart.png
Authentication

API Keys

All requests must include your API key. Three supported methods — the header is preferred.

API V1 — GET

Mini Chart

Returns a compact sparkline-style chart as a PNG image. Best for embedding small thumbnails in bots or alerts.

GET /v1/tradingview/mini-chart Returns PNG directly
Parameters
Name Type Default Description
symbolrequired string — TradingView symbol e.g. BINANCE:BTCUSDT, NSE:RELIANCE
intervaloptional string 1D Timeframe: 1m 5m 15m 1h 4h 1D 1W
widthoptional integer 400 Output width in pixels (max 2048)
heightoptional integer 200 Output height in pixels (max 2048)
keyrequired string — Your API key
cURL
Python
JavaScript
curl "http://localhost:3000/v1/tradingview/mini-chart?symbol=NSE:BANKNIFTY&interval=1D&key=demo-key-basic-001" \
     --output banknifty_mini.png
import requests

resp = requests.get("http://localhost:3000/v1/tradingview/mini-chart", params={
    "symbol": "NSE:BANKNIFTY",
    "interval": "1D",
    "key": "demo-key-basic-001"
})
open("chart.png", "wb").write(resp.content)
const res = await fetch(`http://localhost:3000/v1/tradingview/mini-chart?symbol=NSE:BANKNIFTY&interval=1D&key=demo-key-basic-001`);
const blob = await res.blob();
// Use blob as image source in your bot/app
200 Returns PNG binary — set Content-Type: image/png

TradingView Layout

Layout Render

Capture a screenshot of your own saved TradingView layout — including all custom indicators, drawings, and panel arrangements. The render is processed asynchronously and you poll for the result.

🔗
Get your Layout ID from the share URL: tradingview.com/chart/hMZXawOv/ — the bolded part is your layout ID.
POST /v1/renders Queues a render job → returns job ID
Request Body (JSON)
Field Type Description
enginerequired string Use "tradingview_layout" for layout renders
data.layoutrequired string Your TradingView layout ID (from share URL)
data.symboloptional string Override the symbol shown in the layout
data.intervaloptional string Override the timeframe e.g. "1D"
output.formatoptional string "png" (default) or "jpeg"
output.widthoptional integer Screenshot width (default: 1200)
output.heightoptional integer Screenshot height (default: 630)
cURL
Python
JavaScript
# Step 1: Queue the render
curl -X POST "http://localhost:3000/v1/renders" \
  -H "Authorization: Bearer YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "engine": "tradingview_layout",
    "data": { "layout": "hMZXawOv", "interval": "1D" },
    "output": { "format": "png", "width": 1200, "height": 630 }
  }'
# → {"job_id": "job_abc123...", "status": "queued"}

# Step 2: Poll until done
curl "http://localhost:3000/v1/jobs/job_abc123" \
  -H "Authorization: Bearer YOUR_KEY"
# → {"status": "succeeded", "artifacts": {"cdn_url": "/v1/jobs/job_abc123/result"}}

# Step 3: Download the image
curl "http://localhost:3000/v1/jobs/job_abc123/result" \
  -H "Authorization: Bearer YOUR_KEY" --output layout.png
import requests, time

headers = {"Authorization": "Bearer YOUR_KEY"}
BASE = "http://localhost:3000"

# 1. Queue job
r = requests.post(f"{BASE}/v1/renders", headers=headers, json={
    "engine": "tradingview_layout",
    "data": {"layout": "hMZXawOv", "interval": "1D"},
    "output": {"format": "png", "width": 1200, "height": 630}
})
job_id = r.json()["job_id"]

# 2. Poll
while True:
    status = requests.get(f"{BASE}/v1/jobs/{job_id}", headers=headers).json()
    if status["status"] == "succeeded": break
    time.sleep(2)

# 3. Download
img = requests.get(f"{BASE}{status['artifacts']['cdn_url']}", headers=headers)
open("layout.png", "wb").write(img.content)
const BASE = "http://localhost:3000";
const KEY = "YOUR_KEY";
const headers = { Authorization: `Bearer ${KEY}`, "Content-Type": "application/json" };

// 1. Queue
const { job_id } = await fetch(`${BASE}/v1/renders`, {
  method: "POST", headers,
  body: JSON.stringify({ engine: "tradingview_layout", data: { layout: "hMZXawOv" }, output: { width: 1200, height: 630 } })
}).then(r => r.json());

// 2. Poll every 2s
let job;
do {
  await new Promise(r => setTimeout(r, 2000));
  job = await fetch(`${BASE}/v1/jobs/${job_id}`, { headers }).then(r => r.json());
} while (job.status === "queued" || job.status === "rendering");

// 3. Use the image URL
console.log(`${BASE}${job.artifacts.cdn_url}`);

Job Status

Poll this endpoint after queuing a render job. Typical layout renders complete in 6–15 seconds.

GET /v1/jobs/:job_id 
# Pending
{ "job_id": "job_abc123", "status": "queued" }

# Completed
{
  "job_id": "job_abc123",
  "status": "succeeded",
  "artifacts": {
    "cdn_url": "/v1/jobs/job_abc123/result",
    "expires_at": "2026-04-06T00:00:00Z"
  }
}
200 Job object with status: queued | rendering | succeeded | failed
Reference

Rate Limits

Rate limits are enforced per API key using a token bucket algorithm. When exceeded, a 429 response is returned with a Retry-After header.

⚠️
For layout renders (async), set your timeout to at least 60 seconds when polling. TradingView charts with many indicators can take 10–25 seconds to render fully.
